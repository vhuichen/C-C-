init_py

__init__.py

作用

1. 包的标识

如果一个目录包含 __init__.py 文件，那么这个目录就可以被 Python 解释器识别为一个包。允许使用 import 语句来导入该目录下的模块。

2. 初始化代码

__init__.py 文件可以包含包的初始化代码。当包被导入时，这个文件会被执行，这可以用来初始化包内的变量或者执行一些设置代码

3. 控制可见性

通过在 __init__.py 文件中定义 __all__ 列表，可以控制从包中导入哪些模块或变量

# 这表示只有 module1 和 module2 可以从这个包中被导入
__all__ = ['module1', 'module2']
4. 重命名导入

__init__.py 文件可以用来为包内的模块或变量提供别名，使得它们在导入时使用不同的名字。例如：

from .module import some_function as alias_function
5. 包的元数据

__init__.py 可以包含包的元数据，比如版本号、作者信息等

6. 后向兼容性

在一些旧的 Python 项目中，__init__.py 可能用于确保目录被视为包，即使它们只包含子包。这是在 Python 3.3 之前的做法，因为在 Python 3.3 及以后版本中，即使没有 __init__.py 文件，包含 __pycache__ 目录或 .pyc 文件的目录也会被视为包

7. 避免命名冲突

在某些情况下，__init__.py 可以用于避免命名冲突，比如当你的包名与标准库中的模块名冲突时。

备注：从 Python 3.3 开始，不必在包目录中包含 __init__.py 文件，会自动生成一个文件?

from pkg import module

都可以表示为从某个py文件中导入对应的函数，或者全部导入。

from pkg 本质上也是在找 pkg 对应的__init__.py 文件；

import pkg 相当于导入 pkg 下__init__.py 文件可见的所有函数。

# 表示：从 pkg 包对应的 __init__.py 文件中导入 __all__ 指定的所有文件
from pkg import *

# 表示：从当前包中的 module 子模块中导入 some_function 函数
from .module import some_function
